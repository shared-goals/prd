## Приоритетные задачи

### 🔴 Персонаж — сфокусироваться на одном

Решение принято (апрель 2026): MVP строится вокруг **одного персонажа** — Сергей, 40+, Архитектор. Полная цепочка взаимодействия с платформой реализуется именно для него.

Остальные персонажи (городской волонтёр, борьба с зависимостями) — ценные иллюстрации механики платформы, но **не отдельные юзкейсы для проектирования MVP**. Переносятся в RESEARCH.md как примеры работы Социального капитала.

**Следствия для README/кода:**
- Удалить из требований то, что специфично только для других персонажей
- Онбординг и happy path строить под Архитектора
- Seed network = группа «Шило» (реальные люди с активной жизненной позицией)

### 🟡 Права создателя цели на модерацию коммитов

**Вопрос:** Должен ли создатель public-цели иметь право удалить неуместный коммит участника?

**Варианты:**
- **А.** Нет — создатель не имеет власти над действиями других. Принцип Shared Goals: нет иерархии.
- **Б.** Да, с ограничением — только форматное удаление (спам/флуд), не по содержанию. Решение логируется.
- **В.** Да, с апелляцией — участник может обжаловать через систему модерации.

Предложение: **вариант Б** как минимально достаточный для MVP.

*[на подтверждении у Сергея]*
